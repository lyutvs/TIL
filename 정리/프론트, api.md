# 프론트

*

사용자가 마주보는 인터페이스

매끄럽게 동작하는 자연스러운 프론트 엔드를 구축하는것이 중여

사용자 인터페이스(UI), 사용자 경험(UX)을 만드는데 초점


* UX(user experience)

  사용할때 사용자가 느끼는 느낌, 태도, 행동을 말한다

  ex: 건강관리 어플로 건강관리에 대한 태도가 달라지고, 건강관리 하는 내감정을 변화 시킨다

  특정 사람만 만족시키는 주관성을 지향

* UI(user intface)

  사용자가 제품/서비스 를 사용할떄 마주하게 되는 면

  사용자와 상호작용 할 수 있도록 만들어 진것

  디자인,폰트,컬러,레이아웃등 사용자가 마주하게 되는것

* UI 와 UX 차이점

  UI은 고속도로와 같다 빠르고 안전하게 목저지 도달하게 하지만 아름다운 경치 제공은 안한다

  UX는 드라이브 코스 곡선 도로를 이용하여 시간이 럴리수 있지만 운전자 취향에 따라 산, 바다 등을 볼수 있다


* 매뉴, 레이아웃, 폰트, 컬러등 모든 요소들이 프론트 엔드 개발을 이루는 부분

  구체적인 요구사항, 목표, 기회 등을 이해하고 그에 따른 프론트 엔드를 만들어 낼수 있는 ㅐ발업체와 협력해야 한다.

  인터페이스를 디자인 하기 전에는 목업(mockup), 와이어 프레임(wireframe),클릭할수 있는 프로토 타입(prototype)을 만들어야 한다

* 이런 도구들은 애플리케이션이 사용자 경험안에 있는 이슈들을 알게 된다면 문제를 해결할수 있는것이고, 결국에는 영향력 있느 프론트 엔드를 개발할수 있다

  ​

* 목업(mockup)

  실물과 흡사한 형태의 모형(실물과 비슷하게 화면구성)

  목업종류: 파워목업, 발사믹 목업

* 와이터 프레임(wireframe)

  UI중심의 화면 레이아웃

  손그림, 파워포인트, 카노트,스케치,일러스트,포토샵

  화면 단위 레이아웃 설계하는 작업

  와이어 프레임 툴:미로, 저스트마인드, 어도비, UX핀, 플루이드


* 프로토타입(prototype)

  다양한 이터랙션이 결합되어 실제 서비스처럼 작동하는 모형 툴:카카오 오븐, UX핀,네이버 프로토나우


* 사용언어

    * html

      하어퍼 텍스트오 마크업 언어로 구성되어 있다

    * css

      종속 스타일 시트 (cascading style sheets)의 약자

      개발자가 웹페이지에 다양한 스타일을 적용할 수 있게 해줌으로써 애플리케이션 페이지로 단순하게 만들어 낼수 있다

    * Javascript

      사용자와 상호작용 하는 애플리케이션을 만들수 있도록 도와준다


* 테크놀로지

    * angular Js

      오픈소스 자바스크립트 프레임 워크로써 주로 싱글 페이지 웹 애플리케이션 만드는데 사용

      정적 HTML을 동적 HTML로 변환해주는 기능이 있다

    * React Js

      리액트는 프론트 엔드 개발에서 사용할수 있는 유연하고 효과적인 선언향의 자바스크립트 라이브러리

      뛰어난 사용자 인터페이스를 만드는데 도움이 된다

      애플리케이션의 반응형 뷰레이어

    * Boot strap

      부트스트랩은 반응형 웹 애플리케이션 및 웹사이트를 개발하기 위해 사용되는 오프소스 모료도구

      모바일 우선 반응형 웹사이트 구축 가능

* API


* 애플리케이션 소프트웨어를 구축하고 통합 하기 위한 정의 및 프로토콜 세트로 애플리케이션 프로그래밍 인터페이스를 나타낸다

* 스마트폰과 사용자를 이어주는 매개채 API도 같은 맥락 API는 프로그램과 또 다르 ㄴ프로그램을 연결시켜주는 일종의 다리

* 예시

  롤전적 사이트

  라이엇이 제공하는 open api를 사용해 만드는데 그 API안에 롤유저가 무엇을 했고 게임으 ㄹ이겼는지 졌는지를 담은 데이터를 사용해 롤전적 사이트를 만드는걸 api의 예씨로 들수 있다
